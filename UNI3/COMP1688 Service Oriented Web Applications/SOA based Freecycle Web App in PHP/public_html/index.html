This file is called index.html and is a placeholder for your account on the Apache web (HTTP) server called stuweb.cms.gre.ac.uk provided by the University of Greenwich for students in the Department of Computing. You may find it useful to bookmark this page in your web browser. You are encouraged to edit or replace this file to create your own website. This page is accessible anywhere on the World Wide Web as any of the following URLs

  • http://stuweb.cms.gre.ac.uk/~ab123
  • http://stuweb.cms.gre.ac.uk/~ab123/
  • http://stuweb.cms.gre.ac.uk/~ab123/index.html
  • https://stuweb.cms.gre.ac.uk/~ab123
  • https://stuweb.cms.gre.ac.uk/~ab123/
  • https://stuweb.cms.gre.ac.uk/~ab123/index.html

Replace ab123 with your own userid.

The stuweb.cms.gre.ac.uk web server mounts the public_html directory of your Unix file system provided by the University of Greenwich. The simplest way to access this file system is to use the Unix server called student.cms.gre.ac.uk . Please note that this Unix server is called student.cms.gre.ac.uk whereas the HTTP server is called stuweb.cms.gre.ac.uk. You have SSH and SFTP access to this server from anywhwere on the Internet.

Any secure shell such as PuTTY can be used to log onto to this server. Any FTP client, for example SmartFTP or FileZilla, can be used to transfer files to and from this server. Better still, a text editor with a built-in FTP client, such as Notepad++, Archnophilia or even Netbeans, will enable you to edit your files directly on the server. All of this software can run on a Microsoft Windoze platform and is available free of charge. If you are fortunate enough to be using a Mac or Linux machine you may find that most, if not all of the tools you require to create a website running from stuweb.cms.gre.ac.uk are already installed.

Further guidance on using this web server is available from our Student Technical Support. Please note that file names, file paths and URLs on these servers are all case sensitive. You will have more manageable URLs if you keep your file names short and avoid including spaces in file and directory names.
